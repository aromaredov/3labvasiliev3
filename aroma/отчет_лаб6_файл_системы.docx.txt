Ход выполнения работы

1. Создание структуры каталогов

В домашнем каталоге была создана следующая структура:

~/Васильев/
├── 1/
│   └── group
├── 2/
│   └── group
├── 3/
│   └── group
├── 4/
│   └── group
├── group_hard   (жёсткая ссылка на 1/group)
└── group_sym    (символическая ссылка на 1/group)

Использованные команды:
mkdir ~/Васильев
cd ~/Васильев
mkdir 1 2 3 4


2. Копирование файла /etc/group

Файл /etc/group скопирован в каталоги 1, 2, 3 и 4 с использованием абсолютных имён:

cp /etc/group ~/Васильев/1/
cp /etc/group ~/Васильев/2/
cp /etc/group ~/Васильев/3/
cp /etc/group ~/Васильев/4/


3. Определение типов файлов с помощью утилиты file

Команда:
file /bin/bash /dev/null /etc/passwd ~/Васильев/1/group

Результат:
/bin/bash:                  ELF 64-bit LSB pie executable, x86-64, version 1 (SYSV), dynamically linked, interpreter /lib64/ld-linux-x86-64.so.2, BuildID[sha1]=2f77b36371c214e11670c7d9d92727e9a49f626b, for GNU/Linux 3.2.0, stripped
/dev/null:                  character special (1/3)
/etc/passwd:                ASCII text
/home/aroma/Васильев/1/group: ASCII text

Вывод: утилита file правильно определила типы файлов:
- /bin/bash – исполняемый ELF-файл;
- /dev/null – символьное устройство;
- /etc/passwd – текстовый файл;
- group – текстовый файл.


4. Типы файлов в каталоге /dev

Команда ls -l /dev вывела список файлов. В каталоге /dev присутствуют следующие типы файлов:

| Символ | Тип файла | Пример |
|--------|-----------|--------|
| d      | Каталог   | block, char, disk, pts |
| l      | Символьная ссылка | core -> /proc/kcore, fd -> /proc/self/fd, stdin -> /proc/self/fd/0 |
| c      | Символьное устройство | null, zero, tty, console, random |
| b      | Блочное устройство | sda, sdb, loop0, ram0 |
| s      | Сокет | log (присутствует) |
| p      | Именованный канал | не обнаружен |


5. Изучение ключей утилиты ls (справка man ls)

С помощью команды man ls были изучены следующие ключи:

| Ключ   | Назначение |
|--------|------------|
| -R     | Рекурсивный обход подкаталогов |
| -1     | Вывод в одну колонку (единица) |
| -m     | Вывод через запятую |
| --color| Цветная подсветка типов файлов |
| -t     | Сортировка по времени изменения (новые сверху) |
| -S     | Сортировка по размеру (большие сверху) |
| -X     | Сортировка по расширению |
| -r     | Обратный порядок сортировки |


6. Создание жёсткой и символической ссылок

Для файла ~/Васильев/1/group созданы жёсткая и символическая ссылки:

cd ~/Васильев
ln 1/group group_hard      # жёсткая ссылка
ln -s 1/group group_sym    # символическая ссылка

Проверка ссылок:
ls -li group_hard group_sym 1/group

Результат:
46449 -rw-r--r-- 2 aroma aroma 783 Jun 17 16:04 1/group
46449 -rw-r--r-- 2 aroma aroma 783 Jun 17 16:04 group_hard
46454 lrwxrwxrwx 1 aroma aroma   7 Jun 17 16:05 group_sym -> 1/group

Вывод:
- У файла 1/group и ссылки group_hard одинаковый номер inode (46449) – это подтверждает, что это жёсткая ссылка (одна область данных, два имени).
- У ссылки group_sym другой inode (46454) – это символическая ссылка (отдельный файл, содержащий путь к оригиналу).

Ответы на контрольные вопросы

1. Типы файлов ОС Linux: обычные файлы (-), каталоги (d), символьные ссылки (l), блочные устройства (b), символьные устройства (c), сокеты (s), именованные каналы (p).

2. Назначение утилиты file: определяет тип файла по его содержимому (сигнатурам), не полагаясь на расширение.

3. Структура дерева каталогов ОС Linux: иерархическая, с единым корнем /, от которого отходят все подкаталоги.

4. Отличия файловых систем Windows и Linux: в Linux единый корень / (нет дисков C:), регистрозависимость имён, расширения не обязательны.

5. Отличие каталогов /var и /tmp: /var содержит постоянно изменяемые данные (логи, очереди), /tmp – временные файлы, очищается при загрузке.

6. Назначение утилиты pwd: выводит полный путь текущего каталога.

7. Назначение утилиты cat: выводит содержимое файла на экран (или объединяет несколько файлов).

8. Назначение утилиты ls: вывод списка содержимого каталога. Ключи: -F – добавляет символы типов, -a – показывает скрытые файлы (с .).

9. Утилита mkdir: создаёт новый каталог.

10. Утилиты копирования и перемещения: cp – копирует файлы, mv – перемещает/переименовывает.

11. Жёсткие ссылки: ещё одно имя файла (один inode). Создаются командой ln источник ссылка.

12. Создание файлов: touch имя (пустой файл) или перенаправление > имя.

13. Символьные ссылки: отдельный файл, содержащий путь к другому объекту. Создаются ln -s источник ссылка.

14. Удаление файлов и каталогов: rm – файлы, rmdir – пустые каталоги, rm -r – каталоги с содержимым. Восстановление невозможно (в Linux нет корзины для командной строки).

15. Назначение утилиты id: показывает UID, GID и список групп текущего пользователя.

16. Ярлыки объектов файловой системы (inode): содержат тип, права доступа, владельца, размер, временные метки, указатели на блоки данных.

17. Права доступа к файлу: r (чтение), w (запись), x (выполнение) для трёх категорий: владелец (u), группа (g), остальные (o).

18. Суперпользователь и его права: пользователь root с UID=0, имеет неограниченный доступ ко всем файлам и операциям.

19. Назначение утилиты sudo: позволяет выполнять команды от имени другого пользователя (обычно root) с логированием действий.

20. Утилиты поиска locate и find: locate – быстрый поиск по базе данных (требует updatedb), find – медленнее, но ищет в реальном времени и имеет больше возможностей для фильтрации.
